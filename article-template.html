Machine Reading Comprehension for the Tamil Language using Translated SQuAD

Krishanth G AI & Software Engineer

Developing robust Natural Language Processing (NLP) models for low-resource languages has always been a significant challenge. My recent research focused on building an Intelligent Machine Reading Comprehension (MRC) and Extractive Question Answering (QA) system tailored specifically for the Tamil language.

The Architecture

To decode the complex, agglutinative nature of Tamil script morphosyntactically, I fine-tuned advanced multilingual Transformer models. Specifically, I utilized Google's MuRIL (google/muril-base-cased) and deepset's xlm-roberta-base-squad2.

"By optimizing model hyperparameters, the system was able to effectively understand context-based queries in native Tamil script."

The 512-Token Constraint

One of the largest hurdles in using standard Transformer architectures is the strict 512-token input limit. When evaluating long-form context documents, truncation leads to severe data loss.

To solve this, I engineered a custom Sliding Window Strategy:

  • Chunk Size: 384 tokens
  • Overlapping Stride: 128 tokens
def sliding_window(context, chunk_size=384, stride=128):
    tokens = tokenize(context)
    chunks = []
    for i in range(0, len(tokens), chunk_size - stride):
        chunks.append(tokens[i:i + chunk_size])
    return chunks

This approach allowed the model to seamlessly process entire documents without losing the contextual thread between chunks.

Conclusion

The resulting system successfully extracted answers from complex Tamil paragraphs, proving that with the right data augmentation and windowing techniques, multilingual LLMs can achieve high accuracy on heavily agglutinative scripts.
